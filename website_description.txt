# Don't show the whole phone immediately.

This is what Apple does.

Screen 1

```
        _______

      |       |
      | Hello |
      |       |
      |_______|
```

Then scroll.

Phone slowly grows.

```
          _________

        |         |
        |         |
        |         |
        |_________|
```

Then scroll.

```
                _____________

             |               |
             |               |
             |               |
             |_______________|
```

You're not zooming the user.

You're zooming the product.

# My favorite Apple trick

Instead of making screenshots smaller...

Make **the browser bigger**.

Imagine this:

```
Entire screen

‚Üì

Phone occupies 35%

‚Üì

Scroll

‚Üì

Phone occupies 55%

‚Üì

Scroll

‚Üì

Phone occupies 75%

‚Üì

Scroll

‚Üì

Phone occupies 95%
```

The phone feels like it's coming toward the visitor.

No zoom controls.

No gimmicks.

Just CSS transforms tied to scroll.

---

Bro, if you pull this off, don't think of it as "making a website."

Think of it as **directing a film**.

Every scroll should answer one question:

> **"What does Hermes help me do next?"**

By the end, the visitor shouldn't just know the features. They should understand the philosophy that connects them. That's much closer to how Apple's product pages guide a visitor through a narrative rather than presenting a catalog.
