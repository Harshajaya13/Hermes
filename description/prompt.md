# Hermes Website UI/UX Specification

The content for every section already exists in dedicated Markdown files. Do NOT rewrite, summarize, or hallucinate new content. Render the content exactly as written while presenting it with a premium product showcase similar to modern Apple product pages.

## Overall Philosophy

The website should feel like a guided product experience rather than documentation.

Each scroll should naturally answer one question before moving to the next.

The visitor should never feel lost or wonder what comes next.

The website should tell a continuous story.

---

## Layout

Each major feature occupies approximately one full viewport.

Structure every section as:

Feature Title

↓

Short supporting description

↓

Large phone mockup

↓

Detailed explanation

↓

Subtle transition into the next feature

Do not stack multiple screenshots together.

One screenshot should explain one concept.

---

## Phone Showcase

Every screenshot must be displayed inside a realistic premium phone mockup.

Do not display raw screenshots.

Use:

• Rounded device frame
• Thin bezels
• Soft shadow
• Premium glass appearance

Portrait screenshots should always preserve their aspect ratio.

Never stretch images.

---

## Image Presentation

Images should never appear tiny.

Use approximately 75–90% of the viewport height.

As users scroll:

• Images gently scale
• Images fade in
• Images slightly translate upward

Animation should feel calm and premium.

Avoid flashy animations.

---

## Mobile Optimization

The website must work beautifully on phones.

On mobile:

Title

↓

Description

↓

Phone Screenshot

↓

Detailed Content

Do not place text beside images on small screens.

Everything should become a single vertical story.

Use comfortable spacing.

Text width should remain readable.

---

## Desktop Optimization

Desktop should use a split layout.

Left:

Title

Description

Markdown Content

Right:

Large sticky phone mockup

As the user scrolls:

The phone remains visible.

Only the screenshot changes.

The content scrolls naturally.

This creates a premium showcase similar to Apple's product pages.

---

## Section Navigation

Visitors should never wonder where they are.

Add a floating navigation panel.

Examples:

• Home
• Domains
• Blocks
• Items
• Questions
• Articles
• Notes
• Observations
• Ideas
• Reflections
• Evolutio
• Veritas
• Search
• Manual Collections
• Import
• Export
• Control Center
• Workspaces
• About

Highlight the current section while scrolling.

Allow smooth scrolling to every section.

---

## Reading Progress

Add a thin progress indicator at the top.

It should slowly fill as the visitor progresses through the website.

This helps visitors understand how much remains.

---

## Section Ending

Every section should naturally guide the visitor forward.

At the bottom of each section include a subtle transition such as:

"Next: Blocks →"

or

"Continue to Articles"

Avoid making users search for what comes next.

Scrolling should feel continuous.

---

## Typography

Large bold titles.

Comfortable reading width.

Generous spacing.

Premium typography hierarchy.

Avoid long uninterrupted paragraphs.

Break content into meaningful sections.

---

## Images

Never crop important content.

Allow screenshots to slightly overflow the viewport during scroll animations.

Images should feel like the primary visual element.

The text supports the screenshots rather than competing with them.

---

## Colors

Use a dark premium theme.

Soft grays instead of harsh whites.

Cards should subtly separate from the background.

Avoid bright borders.

Avoid saturated colors unless they already exist inside Hermes.

---

## Motion

Animations should be subtle.

Use:

• Fade
• Scale
• Translate

Avoid:

• Rotations
• Flips
• Excessive parallax
• Fast transitions

Motion should guide attention rather than distract.

---

## Footer

The final section is About Hermes.

After that, display a clean footer with:

GitHub

Latest Release

Documentation

License

Version

End with:

"Information is easy to collect.

Understanding is difficult to preserve.

Hermes exists to bridge the distance between the two."

The visitor should finish the website feeling that Hermes is more than a note-taking app. It should feel like a personal knowledge operating system.
