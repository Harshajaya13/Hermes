# Import Collections

## Purpose

Import Collections allow you to bring large amounts of knowledge into Hermes within seconds.

Instead of manually creating hundreds of Questions or Articles one at a time, you can prepare a structured JSON file and import everything at once.

Hermes validates the file, stores every item locally, and immediately makes it available to the scheduling engine.

Importing knowledge should be a one-time task.

Learning from that knowledge should happen gradually over weeks or months.

---

## Why Import Exists

Building a personal knowledge system should never require repetitive work.

Imagine creating:

- 1,000 Probability Questions
- 500 Linux Articles
- 300 PyTorch Exercises
- 200 Research Papers

Creating each item manually would take hours.

Importing allows you to prepare the knowledge once and let Hermes organize the rest.

This keeps your focus on learning rather than data entry.

---

## Preparing Your Collection

Hermes accepts structured JSON files.

Each file contains a collection of either Questions or Articles.

Before importing, simply prepare the file using your preferred editor or generate it using AI.

Once ready, Hermes validates every item before saving anything into the database.

If the file contains errors, Hermes will explain the problem instead of importing incomplete or invalid data.

---

## Import Workflow

Importing follows a simple process.

Choose Collection

↓

Select JSON File

↓

Validate Structure

↓

Preview Import

↓

Import into Local Database

↓

Ready for Scheduling

Nothing is immediately added to Today's Pursuit.

Importing only stores knowledge.

Scheduling happens separately according to your Daily Limits.

---

## Local First

Every imported item is stored locally.

Your Questions and Articles remain available even without an internet connection.

Hermes never depends on an external server to preserve your knowledge.

Once imported, your collection belongs entirely to you.

---

## Safe Validation

Before importing, Hermes performs validation to ensure the file follows the expected structure.

Validation checks:

- Required fields
- Item type
- Collection format
- Missing values
- Invalid JSON

This prevents broken imports and keeps your knowledge base consistent.

---

## Growing Over Time

Importing is not something you'll do every day.

Instead, you'll gradually build larger collections over months.

Perhaps today you import:

- Probability Questions

Next week:

- Linux Articles

Later:

- Research Papers

Your library continues growing while Hermes quietly schedules everything in the background.

---

## Philosophy

Importing knowledge should feel effortless.

The effort should go into understanding the knowledge, not typing it into the application.

Hermes removes repetitive work so your attention stays where it belongs:

Learning.
