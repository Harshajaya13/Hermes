Think of it like this.

Product Philosophy
        ↓
Information Architecture
        ↓
UX Architecture
        ↓
Design System
        ↓
Flutter Code

So UX Architecture is the bridge between the structure and the visuals.
I think this document should answer only interaction questions.

Not colors.

Not spacing.

Not fonts.

Instead...


UX Principles

These are non-negotiable.

For example

1.
One purpose per screen.

2.
Never overwhelm the user.

3.
Every interaction must reduce friction.

4.
Every action should feel reversible.

5.
Teaching should happen naturally.

6.
The interface should disappear behind the user's thinking.

7.
The app should feel calm, not urgent.

8.
Growth is more important than completion.

9.
Empty states should educate, never punish.

10.
The fastest path should also be the best path.

I also want to introduce something.

I call it...

The Silence Principle.

Most apps constantly ask for attention.

Hermes shouldn't.

For example.

❌ Popups.

❌ Rate us.

❌ Daily streak.

❌ Claim reward.

❌ Spin wheel.

❌ Achievement unlocked.

Instead...

Hermes quietly waits.

When you return...

It simply says

Welcome back.

Where shall we continue?

That feels respectful.

I think this might become my favorite sentence in the whole Codex.

Hermes never competes for the user's attention. Hermes protects it.

I think that sentence belongs in both the Product Philosophy and the UX Architecture.

Because every screen, every interaction, and every animation should be judged against that one principle.

If an interaction steals attention instead of protecting it, it doesn't belong in Hermes. I think that's one of the strongest UX principles we've uncovered so far, and it perfectly matches the philosophy you've been building from the beginning.
