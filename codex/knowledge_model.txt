# 06_Data_Model.md

## Purpose

The Data Model defines how every object inside Hermes is stored, connected, backed up, restored and moved between devices.

It is responsible for preserving the user's journey, not just application data.

---

# Core Principles

* Offline First
* User Owns Their Data
* Local Storage by Default
* Portable Workspaces
* Reversible Actions
* Privacy by Choice
* Future Compatible

---

# Object Relationships

Workspace
└── Domain
└── Block
└── Item
└── Reflection
└── Evolutio

Evolution is **not stored**.

Evolution is generated dynamically from every Evolutio.

---

# Identity

Every object has a permanent unique identifier.

Workspace

* workspace_id

Domain

* domain_id

Block

* block_id

Item

* item_id

Reflection

* reflection_id

Evolutio

* evolutio_id

Veritas

* veritas_id

---

# Metadata

Every object stores:

* id
* created_at
* modified_at
* archived
* deleted
* version

Optional:

* icon
* color
* description

---

# Workspace

Contains:

* Domains
* Settings
* Archive
* Privacy Settings

Properties:

* Name
* Description
* Icon
* Visibility
* Privacy
* Theme Override (optional)

Capabilities:

* Rename
* Export
* Import
* Backup
* Restore
* Lock
* Hide
* Archive

---

# Workspace Privacy

Visibility

* Public
* Private

Private Workspace

* Hidden from Home
* Requires PIN or Pattern
* Can optionally enable Encryption

Encryption is optional.

It is never forced.

---

# Domain

Purpose

Represents one intentional area of growth.

Examples

* Engineering
* Thinking
* Business
* Life

Capabilities

* Rename
* Archive
* Export
* Import
* Reorder
* Lock (optional)

Contains

Multiple Blocks.

---

# Block

Purpose

Interactive environment dedicated to one specific area of growth.

Examples

* Mathematics
* Psychology
* Startup
* Python

Capabilities

* Rename
* Pin
* Favorite
* Archive
* Export
* Import
* Change Color
* Change Icon

Contains

Items.

---

# Item

Supported Types

* Question
* Article
* Reflection Note
* Quote
* Observation
* Idea

Items intentionally do not contain:

* Videos
* PDFs

Those remain external resources.

---

# Reflection

Purpose

Capture the user's own thinking.

Rules

* Required for meaningful growth.
* Multiple reflections allowed.
* Editable.
* Timestamp preserved.

Reflection belongs to an Item.

---

# Evolutio

Purpose

Represents one meaningful recorded change in understanding.

Rules

* Never based on streaks.
* Never based on points.
* Never compared with others.
* Created after meaningful reflection.
* User driven.

An Evolutio is evidence of growth.

Not a score.

---

# Evolution

Not stored.

Generated dynamically.

Evolution = Sum of all Evolutios over time.

---

# Veritas

Independent object.

Purpose

Record honest reasons for missed days.

Rules

* Optional
* Searchable only when enabled
* Appears on empty calendar days
* Never judged

---

# Archive

Acts like a filesystem archive.

Deleting a parent object moves every child object with it.

Example

Deleting a Workspace archives:

* Domains
* Blocks
* Items
* Reflections
* Evolutios

Restoring restores the entire hierarchy.

Permanent deletion is available only from Archive.

---

# Search Index

Search includes

* Domains
* Blocks
* Questions
* Articles
* Reflections
* Quotes
* Evolutios

Veritas is excluded unless enabled.

---

# Import & Export

Hermes uses a portable workspace format.

Extension

.hermes

A .hermes package contains

* Metadata
* Workspace Information
* Domains
* Blocks
* Items
* Reflections
* Evolutios
* Icons
* Settings
* Archive
* Manifest

---

# Package Metadata

Every .hermes package stores

* Schema Version
* Hermes Version
* Created Date
* Modified Date
* Description
* Optional Author
* Checksum

This guarantees compatibility across future Hermes versions.

---

# Ownership Principle

The user owns every part of their journey.

Hermes must make it easy to:

* Protect
* Move
* Backup
* Restore
* Archive
* Share

without depending on any online service.
