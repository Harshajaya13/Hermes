# 📜 The Laws of Hermes

These eight laws dictate all architectural, feature, and design decisions. Any proposal that violates even a single law must be rejected immediately.

---

## 🏛️ The Eight Laws

### ⚖️ Law 1: Intentionality
> **Nothing enters Hermes accidentally. Every item is intentionally chosen.**
> * No feeds, no auto-generated recommendations, and no auto-imported lists. The user must manually and deliberately curate their workspace.
> * **Importing knowledge should be as intentional as creating it.** Hermes should never assume the user wants everything inside a bundle. Users remain in complete control of what enters their workspace.

### 🔇 Law 2: Attention Respect
> **Hermes never manipulates attention.**
> * No push notifications, no badges, no red dots, and no hooks designed to bring the user back. The user controls when they interact with the system.

### 🔌 Law 3: Local Sovereignty
> **Offline is the default.**
> * The system must be fully functional offline. The network is strictly an external resource, never a core requirement.

### 🛡️ Law 4: Non-Judgmental Ownership
> **The user owns their journey. Hermes never judges it.**
> * The system records truth, not compliance. No red markers for "missed" days, no warnings, and no lectures.

### 🕊️ Law 5: Anti-Guilt Reflection
> **Reflection is encouraged. Guilt is forbidden.**
> * Designed to foster understanding. Tools like *Veritas* exist to document context and reasons, replacing shame with honest insight.

### ⚡ Law 6: Performance as a Feature
> **Performance is a feature.**
> * Zero lag, zero bloat, instant startup. The interface must be responsive enough to disappear into the background.

### 🧼 Law 7: Friction Reduction
> **Every screen must reduce friction.**
> * Eliminate unnecessary steps, transitions, and redundant options. Every page is streamlined to support immediate focus.

### 🌱 Law 8: Evolution Over Completion
> **One Evolutio is worth more than one hundred unfinished tasks.**
> * Value deep cognitive shifts over shallow checklists. Focus on long-term personal change, not high-volume execution.

---

## ⏱️ The 3-Second Rule

When opening any screen in Hermes, the user should be able to answer three questions within **3 seconds**:

1. **Where they are.**
2. **Why they are there.**
3. **What they can do next.**

*If a screen fails this test, it requires an immediate redesign.*
